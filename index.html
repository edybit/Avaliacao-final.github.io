Comandos Git

O que é o GIT?

O git é um sistema de controle de versão de códigos. Gerencia ramificações feitas por diferentes pessoas de um time de modo a manter a ordem e integração de todas as partes. Ou seja, mesmo que várias pessoas estejam trabalhando em um programa ao mesmo tempo, ainda é possível controlar as mudanças para evitar erros.

Além disso, o git é um sistema que permite rastrear alterações e manter um histórico dessas alterações para possibilitar voltar a uma versão estável sempre que ocorrer um problema. Ou seja, caso algum erro quebre o sistema em desenvolvimento, é viável voltar para a versão funcional saudável.

Para facilitar o uso e entendimento dessa ferramenta, abaixo segue uma lista das principais coisas que você precisa saber para poder utilizar dessa ferramenta:

  • Repositório: São espaços criados para armazenar seus códigos
  • Branch: É o nome dado a uma versão/ramificação do projeto
  • Merge: Comando responsável por unir modificações para a branch principal

Comandos Git fundamentais

Agora que conhecemos alguns termos a respeito do GIT, que tal aprendermos seus comandos mais básicos?

  • Init: Esse comando cria um repositório novo, seja ele remoto ou local, ou reinicializa um repositório já existente;
  • Clone: Como o nome sugere, este comando clona o código do repositório para manipulação em outro ambiente;
  • Commit: Este comando move os arquivos da State Area para um repositório local;
  • Add: Já este comando, como sugere seu nome, adiciona um arquivo para a chamada Staging Area, ou seja, o prepara para um Commit;
  • Push: Envia os arquivos do repositório local para o remoto;
  • Pull: Este faz o contrário do Push, ele move arquivos de repositórios remotos para locais;
  • Merge: Serve para unir arquivos alterados ao arquivo original, básicamente unir os commits com a branch principal, na maioria dos casos;
  • Log: Permite ver o histórico de commits de um arquivo ou usuário;

Comandos Git Avançados

Para executar esses comandos git, basta abrir o prompt de comando ou CMD no seu computador.

  • git revert: O revert é um dos comandos git aplicados para garantir a segurança dos nossos projetos. Permite desfazer algum commit e recuperar uma versão saudável;
  • git checkout: Para fazer o merge corretamente, é preciso olhar esse outro comando, o checkout. O objetivo dele é fazer a pessoa programadora mudar de branch;
  • git rm: O git rm é um comando muito útil para remover arquivos do git e parar de monitorá-los, ou seja, de associá-los ao repositório;
  • git stash: O stash serve para criar uma pilha de alterações que serão enviadas posteriormente para o repositório. É uma boa forma de guardar algumas mudanças em espera enquanto você muda de branch para trabalhar em outros aspectos do sistema
  • git config: O config é um comando inicial para vincular o trabalho no repositório com sua conta no github. Assim, é configurado com o nome e com o e-mail;
